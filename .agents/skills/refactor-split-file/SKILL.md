# Refactor: Split File into Directory

## 适用场景
当用户要求重构指定文件，将其拆分为多个文件到一个目录中时使用此 skill。

## 工作流程

### PHASE 1: 分析源文件
1. 读取指定文件的内容
2. 分析文件结构，识别可拆分的模块、类、函数、组件等
3. 规划拆分方案：确定拆分为哪些文件，每个文件包含什么内容

### PHASE 2: 创建目标目录
1. 在与源文件同级的位置创建新目录（目录名通常为源文件去掉扩展名）
2. 如果源文件有对应的测试文件，考虑一并处理

### PHASE 3: 创建拆分后的文件
1. 根据规划创建各个新文件，确保每个文件不超过300行
2. 保持原有的导入/导出关系
3. 更新各文件之间的导入路径

### PHASE 4: 验证和清理
1. 验证拆分后的代码能够正常工作
2. 删除源文件
3. 运行项目的类型检查和 lint（如果项目有配置）

## 注意事项
- 每个文件不超过300行
- 保持代码的逻辑完整性和可读性
- 确保导入路径正确
- 删除源文件
- 遵循项目的代码风格和约定
